Proposal

Summery

We are going to develop an API for Android applications to take advantage of SIMD on ARM processors, in which NEON provides a set of scalar/vector instructions and registers. We use Android Studio as platform and Android NDK to assist the implementation. The API includes the basic wrapper of Neon instruction, parallel algorithms, and/or parallel data structure.

Background

There are several common parallelization methods in android programming, including - Using class Thread, which allows running tasks in a separate thread; - Using class AsyncTask, which allows running tasks in a separate thread, too.

However, all of these method only make use of multithreads but not take advantage of the lower level SIMD. As far as we know, there are no tool/library that can be used in developing android apps to access SIMD in arm processor.

In C/C++ world, NEON provides a set of scalar/vector instructions and registers (shared with the FPU) comparable to MMX/SSE/3DNow! in the x86 world. However, android developer must make use of the Native Development Kit (NDK) to embed components that make use of C/C++ code in their Android applications. We want to develop a java library (including the high level wrapper of scalar/vector instructions and several useful functions) for general android developer so that they can take advantage of SIMD mechanism in arm processor without writing C/C++ code.

The Challenge

The first challenge is to get familiar with Android platform and build a stable platform for us to call the functions written in C/C++ code. It is hard for us because we do not know much about Android programming with C/C++. The platform also depend on the model of core. We think this part may need about 3 days work to fully understand the platform.

Then we need to choose and learn some basic algorithms (general propose algorithms as well as image processing algorithms) and use NEON to implement these basic functions for Android platform. In this part, we need to implement parallel algorithms in SIMD and wrap them and create java interface. Only part of algorithms can be writen in SIMD so we will try to discover some. This is the main part of our project. The workload of this part depends on the algorithms we choose. We should arrange about two weeks on this part.

After that, we need to pack the interface into library for other android user to use. We did not have experience in this area so it is hard for us evaluate its workload. We hope that we can finish this part in 4-6 days.

Finally, we should optimize our implementation (eg, improve its robustness for different platform), write some sample code using these API and verified its correctness as well as effiency. This may cause about 4-6 days.

Resources

We current have sample code of ndk, sample code of neon, and a couple some research papers and blogs (see REFERENCE). We should start from the scratch. Besides, we need one android smart phone for our testing.

Goals and Deliverable

PLAN TO ACHIEVE: We plan to develop an some parallel algorithms, such as scanning, sorting, fft and so on. Then we should wrap them as API provided for at least one special machine. The performance provided by these API should outperform the serials and even the multi process version of code.

EXTRA GOALS: Explore more algorithms or implement some parallel data structure. Improve our codes' robustness for different hardware. Keep improve its performance and find which implementation is better.

OTHERS: We will implement an android application using our API and without our API. We hope our API will be convenient for use and perform well.
